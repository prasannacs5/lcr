# LCR - Liquidity Coverage Ratio Monitor

Aura Bank Intraday Liquidity Coverage Metrics and Analysis. Python FastAPI backend + React frontend, deployable as a Databricks App on any workspace.

## Architecture

```
Databricks App (FastAPI)
  |-- /api/results, /api/hqla, ...   --> SQL Warehouse (cfo.aura_bank.*)
  |-- /api/executive-summary         --> Foundation Model (Gemini) for summarization
  |-- /api/chat                      --> Model Serving Endpoint (LCR Agent)
  |-- /                              --> React SPA (frontend/dist)

LCR Agent (Model Serving Endpoint)
  |-- MCP Genie Room                 --> Natural language queries over LCR tables
```

## Prerequisites

- Databricks workspace with Unity Catalog enabled
- SQL Warehouse (serverless or pro)
- Tables in `cfo.aura_bank` schema (see [Data Setup](#data-setup))
- A Genie AI/BI room configured over the LCR tables
- A chat-capable foundation model endpoint (e.g. `databricks-gemini-2-5-pro`)

## Deploy to a New Databricks Environment

### Step 1: Prepare the Data

The app expects these tables under `cfo.aura_bank` (configurable via `LCR_CATALOG` / `LCR_SCHEMA`):

| Table | Description |
|-------|-------------|
| `lcr_results` | LCR calculation results with ratios and dates |
| `hqla_breakdown` | High Quality Liquid Assets breakdown |
| `cash_outflows_breakdown` | Net cash outflows detail |
| `general_ledger` | General ledger entries |
| `reco_table` | Executive summary text (generated by `generate_LCR_recommendations` notebook) |
| `haircuts` | Asset haircut rates |
| `runoff_rates` | Liability runoff rates |

If migrating from another workspace, export/import these tables. Then run the `notebook/hydrate.py` and `notebook/data_prep.py` notebooks to initialize data if starting fresh.

### Step 2: Create a Genie AI/BI Room

1. In the target workspace, go to **AI/BI > Genie Rooms** and create a new room
2. Add the `cfo.aura_bank` tables as data sources
3. Note the **Genie Room ID** from the URL (the hex string after `/genie/`)

### Step 3: Create a Secret Scope for Agent Credentials

The LCR agent serving endpoint needs a service principal with access to the Genie room and SQL warehouse.

```bash
# Create a service principal
databricks service-principals create --display-name "lcr-agent-sp" --profile=<your-profile>

# Create OAuth secret for the SP
databricks api post /api/2.0/accounts/servicePrincipals/<sp-id>/credentials/secrets \
  --profile=<your-profile> --json '{}'

# Create secret scope and store credentials
databricks secrets create-scope lcr-agent --profile=<your-profile>
databricks secrets put-secret lcr-agent databricks_host \
  --string-value "https://<workspace>.cloud.databricks.com" --profile=<your-profile>
databricks secrets put-secret lcr-agent client_id \
  --string-value "<sp-application-id>" --profile=<your-profile>
databricks secrets put-secret lcr-agent client_secret \
  --string-value "<sp-oauth-secret>" --profile=<your-profile>
```

Grant the SP permissions:

```sql
GRANT USE_CATALOG ON CATALOG cfo TO `<sp-application-id>`;
GRANT USE_SCHEMA ON SCHEMA cfo.aura_bank TO `<sp-application-id>`;
GRANT SELECT ON SCHEMA cfo.aura_bank TO `<sp-application-id>`;
```

```bash
# Grant CAN_USE on SQL warehouse
databricks api patch /api/2.0/permissions/sql/warehouses/<warehouse-id> \
  --json '{"access_control_list":[{"service_principal_name":"<sp-application-id>","permission_level":"CAN_USE"}]}'

# Grant CAN_RUN on Genie room
databricks api patch /api/2.0/permissions/genie/<genie-room-id> \
  --json '{"access_control_list":[{"service_principal_name":"<sp-application-id>","permission_level":"CAN_RUN"}]}'
```

### Step 4: Register and Deploy the Agent Model

1. Upload `notebook/agent.py` and `notebook/LCR_Deploy.py` to the workspace:

```bash
databricks workspace mkdirs /Users/<you>/lcr/notebook --profile=<your-profile>
databricks workspace import /Users/<you>/lcr/notebook/agent.py \
  --file notebook/agent.py --format AUTO --profile=<your-profile>
databricks workspace import /Users/<you>/lcr/notebook/LCR_Deploy \
  --file notebook/LCR_Deploy.py --language PYTHON --profile=<your-profile>
```

2. Run the `LCR_Deploy` notebook with these widget values:

| Widget | Value |
|--------|-------|
| `genie_room_id` | Your Genie Room ID from Step 2 |
| `secret_scope` | `lcr-agent` (or your scope name) |
| `llm_endpoint_name` | `databricks-claude-3-7-sonnet` (or any chat model) |
| `endpoint_name` | `lcr-agent-endpoint-v4` |
| `catalog` | `cfo` |
| `schema` | `aura_bank` |

The notebook will register the model in Unity Catalog and create/update the serving endpoint.

### Step 5: Generate Executive Summary Data

Run the `notebook/generate_LCR_recommendations.py` notebook. Set the `DATABRICKS_SUMMARY_MODEL` environment variable if the default model (`databricks-gemini-2-5-pro`) is not available:

```python
import os
os.environ["DATABRICKS_SUMMARY_MODEL"] = "databricks-gemini-2-5-flash"  # or another model
```

### Step 6: Deploy the Databricks App

1. Edit `app.yaml` and set the environment-specific values:

```yaml
env:
  - name: DATABRICKS_WAREHOUSE_ID
    value: "<your-sql-warehouse-id>"       # Required
  - name: LCR_CHAT_ENDPOINT_NAME
    value: "lcr-agent-endpoint-v4"         # Must match Step 4
  - name: DATABRICKS_SUMMARY_MODEL
    value: "databricks-gemini-2-5-pro"     # Must exist on workspace
```

2. Build the frontend (if not already built):

```bash
cd frontend && npm install && npm run build && cd ..
```

3. Upload and deploy:

```bash
# Upload app files to workspace
databricks workspace mkdirs /Users/<you>/lcr/app --profile=<your-profile>
databricks workspace import-dir . /Users/<you>/lcr/app --overwrite --profile=<your-profile> \
  --exclude-dir=node_modules --exclude-dir=.git --exclude-dir=notebook

# Create the app
databricks apps create lcr --profile=<your-profile>

# Deploy
databricks apps deploy lcr \
  --source-code-path /Workspace/Users/<you>/lcr/app \
  --profile=<your-profile>
```

4. Grant the app's service principal access to resources:

```bash
# Get the app SP client ID
databricks apps get lcr --profile=<your-profile>
# Look for service_principal_client_id in the output

# Grant permissions (same SQL + API calls as Step 3, using the app SP)
# Also grant CAN_QUERY on the serving endpoint:
databricks api patch /api/2.0/permissions/serving-endpoints/<endpoint-id> \
  --json '{"access_control_list":[{"service_principal_name":"<app-sp-id>","permission_level":"CAN_QUERY"}]}'
```

## Configuration Reference

### App Environment Variables (`app.yaml`)

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABRICKS_WAREHOUSE_ID` | Yes | - | SQL warehouse ID for data queries |
| `LCR_CHAT_ENDPOINT_NAME` | No | `lcr-agent-endpoint-v4` | Model serving endpoint name |
| `DATABRICKS_SUMMARY_MODEL` | No | `databricks-gemini-2-5-pro` | Foundation model for summaries |
| `LCR_CATALOG` | No | `cfo` | Unity Catalog name |
| `LCR_SCHEMA` | No | `aura_bank` | Schema name |
| `LCR_CHAT_SYSTEM_CONTEXT` | No | *(empty)* | System prompt for chat |
| `LCR_CHAT_MAX_TURNS` | No | `25` | Max agent tool+model turns |
| `LCR_CHAT_STREAM` | No | `true` | Enable streaming chat |

### Agent Serving Endpoint Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `LCR_GENIE_ROOM_ID` | Yes | - | Genie AI/BI room ID |
| `LCR_LLM_ENDPOINT_NAME` | No | `databricks-claude-3-7-sonnet` | LLM for the agent |
| `DATABRICKS_HOST` | Yes | - | Workspace URL (use `{{secrets/...}}`) |
| `DATABRICKS_CLIENT_ID` | Yes | - | SP client ID (use `{{secrets/...}}`) |
| `DATABRICKS_CLIENT_SECRET` | Yes | - | SP secret (use `{{secrets/...}}`) |

## Local Development

### Backend

```bash
cp .env.example .env   # Edit with your values
pip install -r requirements.txt
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The Vite dev server proxies `/api` to `http://localhost:8000`.

## Build Frontend for Deployment

```bash
cd frontend && npm run build
```

Output goes to `frontend/dist/`. The FastAPI app serves it at `/`.
